#### <:이란?!>

> : (콜론)이 있으면 따옴표 안을 "변수(데이터)"로 계산하고, 없으면 그냥 "글자(문자열)"로 취급

- :checked="checked" : checked라는 변수가 true인지 false인지 확인해서 체크박스를 켰다 껐다 함.
-> 내가 만든 태크 안에 :넣는 거는 prop로 보낼 것들을 적어주는 것

- checked="checked" : 그냥 checked라는 글자를 써넣은 것뿐이라, 상태가 변해도 체크박스는 꿈쩍도 안 함.

- 원래 HTML에 있던 속성인데, 그 안에 들어갈 내용을 자바스크립트 변수로 갈아 끼우고 싶을 때 앞에 :를 붙인다!

#### style module

> 1. 여러 파일에서 같이 class = child라고 설정했을 경우 스타일 지정할 때 이름이 같아서 힘들다.
2. style태그에 module을 적어주면 스타일을 독립된 하나의 객체로 인식한다. 
3. 그때 Vue내부에서는 Child3_child_x7y8z라는 암호가 생긴다.
4. 내가 이거를 알 수 없으니 그거를 지명하는 $를 붙여 $style으로 활용한다.

3. 모듈'은 "독립되어 있고, 필요할 때 가져다 쓰는 부품"을 의미

4. $emit('check-changed', { id, checked: ... }) --> CheckBox1 @check-changed="CheckBoxChanged" ... />   
자식이 emit을 하면 부모가 세트로 받아준다.

     위치                   코드                         역할
자식 (보낼 때)     "$emit('check-changed', ...)"       신호 발생 (발신기)
부모 (받을 때)      "@check-changed=""...""",          신호 대기 (수신기)
부모 (함수)          CheckBoxChanged(e) { ... },       실제 작업 (처리반)
