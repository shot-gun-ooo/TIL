### ✅ 오늘 학습한 핵심 내용

#### <v-model의 내부 원리>
> `v-model`은 사실 아래 두 가지 기능을 합친 것이다.

1. `v-bind :value`: 데이터 -> 화면

2. `v-on input` : 화면 -> 데이터

`입력을 받을 때 특수한 처리(글자 수 제한, 형식 검사 등)가 필요하다면 v-model 대신 이렇게 찢어서 작성한다.`

---

#### <e.target.value>

- `e` (event): 발생한 사건의 정보 주머니.
- `e.target`: 사건이 일어난 대상 (여기서는 input 태그).
- `e.target.value`: 사용자가 지금 막 입력한 글자 값.
- **결론**: 사용자가 치는 글자를 실시간으로 가로채서 데이터 창고(`this.name`)에 쑤셔 넣는 과정이다.

---

#### <삼항 연산자>

> 조건 ? 참일 때 값 : 거짓일 때 값

- 항이 3개 : 조건, 참일 때 값, 거짓일 때 값
- ex) let result = isMyLayout ? myLayout : '';

---

#### <클래스>

> 1. class 본질 : 별명 붙이기

- HTML 태그한테 class="aaa"라고 하는 건,"너는 이제부터 'aaa'라는 별명을 가진 녀석이야!" 라고 선언하는 것이다.
- 얘를 부를 때 점(.)을 찍는다.

> 2.  Vue에서의 객체 모드 { }
>     > :class="{ '별명' : 조건 }"

- 조건이 true면 별명이 붙고, false면 별명이 떨어진다.

> 3.  Vue에서의 배열 모드 [ ]
>     > :class="[변수1, 변수2]"

- 기본 디자인 + 조건부 디자인
- ex) <div :class="{ 'red-text': hasError, 'bold-text': isImportant }"></div>

`HTML 태그에서 class를 쓸 경우에는 태그 내에서 먼저 정의하고, Vue의 경우에는 .클래스명 으로 스타일을 먼저 지정한다`

### 🔥 알게된 점

1. input 입력값을 숫자로 바꾸어줄 때,VM에서 숫자로 미리 변환해도 HTML 입력창을 거치면 글자로 변한다. 그래서 .number를 꼭 v-지시자에서 해주어야 한다.

2. v-on:으로 시작하는 모든 이벤트는 @로 바꿀 수 있다.

3. v-on에서 함수 호출 시 괄호()를 생략하는 이유

    @click="ctxStop": 이벤트 객체(e)를 인자로 자동 넘겨줌.

    @click="ctxStop()": 인자를 넘기지 않고 함수만 단순 실행함.

4. confirm의 정체 (함수의 역할)
    confirm("메시지")를 실행하면 화면 중앙에 작은 창이 뜨고, 사용자가 확인 누르면 true, 취소 누르면 false

### ⚠️ 오답 노트

1. `v-on` 뒤에는 반드시 콜론(`:`)이 붙거나 `@`를 써야 한다. (예: `@click`)
2. 객체 내부 설정(`name`, `data`) 시에는 `=`이 아니라 `:`를 사용한다.
3. `v-model.number`를 쓰면 클릭 이벤트 로직에서 다음에 나오는 변수에 `parseInt()`를 생략할 수 있어 코드가 간결해진다.
